## 1. 서비스 개요

이터널 리턴(Eternal Return) 게임의 메타 분석 서비스. 랭크 게임 데이터를 자동으로 수집하여 캐릭터 통계, 패치별 트렌드, 파티 조합 추천을 제공한다.

### 구성

| 레포 | 역할 | 기술 스택 |
|------|------|-----------|
| ERMeta | 사용자 대면 프론트엔드 | Next.js 16 App Router, Tailwind v4 |
| ERmangho | 데이터 수집 + API | Next.js + Prisma + Supabase(PostgreSQL) |
| ERMeta/desktop | Windows 전용 클라이언트 | Electron (Steam 로그인 + 로그 테일링) |

---

## 2. 현재 구현된 핵심 기능

### 2-1. 메인 대시보드
- 패치 버전 / 티어 필터 (다이아, 메테오라이트, 미스릴, 1000위 이내)
- 캐릭터 티어 랭킹 테이블 (승률 기반 S/A/B/C/D)
- 트렌딩 섹션 (직전 패치 대비 상승/하락 캐릭터)

### 2-2. 캐릭터 분석 페이지
- 전체 캐릭터 목록 + 이름 검색
- 패치 비교 차트 (승률 / 평균 RP 트렌드, 최근 10패치)
- 패치 내역 탭 (버프/너프/변경 분류)
- 아이템 통계 탭
- 상세분석 탭
- 무기군별 세분화 (캐릭터별 최적 무기군 선택 가능)

### 2-3. 시너지 페이지 (파티 조합)
- 아군 1~2명 선택 → 최적 3번째 캐릭터 추천
- 정렬 기준: 평균 RP / 승률 / 게임 수
- 관심 캐릭터 필터 (2인 고정 시 3번째를 원하는 캐릭터군으로 제한)
- 조합 중복 제거 로직 (조합 순서 무관)

### 2-4. 데스크탑 앱 (Electron)
- Steam OpenID 로그인
- Player.log 실시간 테일링 → 현재 매칭 캐릭터/파티 자동 인식
- 조합 추천 실시간 조회

### 2-5. 데이터 수집 (ERmangho)
- Vercel Cron 5분 주기 수집
- BSER API → 랭크 게임 데이터 파싱
- CharacterStats / CharacterTrio / CharacterTrioByWeapon 원자적 upsert
- 패치별 통계 분리 (CharacterStats), 2주 rolling window (Trio)

## 3. 서비스 방향성 평가

### 강점

**데이터 깊이**: 단순 승률 외에 평균 RP, 평균 순위, 무기군별 세분화, 패치 히스토리까지 제공한다. 경쟁 서비스 대비 분석 깊이가 높다.

**조합 추천의 실용성**: 아군 2명을 고정하고 3번째를 추천받는 인터페이스는 실제 게임 중 발생하는 의사결정 흐름과 정확히 일치한다.

**데스크탑 앱의 차별성**: Player.log 테일링 기반 실시간 인식은 웹서비스로는 불가능한 기능이다. 경쟁 서비스와 명확한 차별점.

**기술 부채 해결**: 이번 세션에서 atomic upsert + rankSum으로 race condition과 float 오차를 모두 해결했다.

## 4. 미흡점 상세 분석

### [P0] 데이터 정확도 - pickRate 항상 0

**현상**: CharacterStats의 `pickRate` 필드가 항상 0으로 저장됨. 수집 로직에서 전체 게임 수 대비 해당 캐릭터 등장 횟수를 계산하지 않고 있음.

**영향**: 픽률은 메타 분석의 기본 지표. "강하지만 안 쓰이는 캐릭터" vs "많이 쓰이는 약한 캐릭터"를 구분할 수 없음.

**해결 방향**: 수집 배치 내 전체 게임 수를 추적하고 캐릭터 등장 비율로 계산 후 별도 upsert.

---

### [P0] Trio 데이터 오염 - TTL 없음

**현상**: CharacterTrio / CharacterTrioByWeapon는 `patchVersion` 없이 누적됨 (설계 의도: 모수 확보). 수집은 최근 2주 게임만 대상으로 하지만, DB 테이블 자체는 전혀 정리되지 않음.

**영향**: 6개월 전 메타의 조합 데이터가 현재 추천에 섞임. 오래된 강조합이 높은 게임 수로 현재 메타를 왜곡.

**해결 방향**:
- 옵션 A: 주기적 정리 (Cron으로 `totalGames`가 낮거나 `updatedAt`이 오래된 row 삭제)
- 옵션 B: `updatedAt` 인덱스 + 조회 시 2주 이내 row만 반환
- 권장: 옵션 B (데이터 보존 + 쿼리 필터 조합)

---

### [P1] 캐릭터/패치 노트 하드코딩

**현상**:
- `characterMap.ts`: 캐릭터 코드 1~86 하드코딩 배열
- `patch-notes/`: 각 패치별 버프/너프 내역이 TypeScript 파일로 수동 관리

**영향**:
- 신규 캐릭터 추가 시 배포 필요 (BSER API에서 자동으로 가져올 수 있음)
- 패치 노트 업데이트 시 개발자가 직접 코드 수정 → 업데이트 지연

**해결 방향**:
- 캐릭터 목록: BSER l10n API 활용 (이미 `/api/bser/l10n/[language]` 존재)
- 패치 노트: 관리자 CMS 또는 마크다운 파일 + GitHub Actions 자동화

---

### [P1] 모바일 미지원

**현상**: UI가 고정 너비 레이아웃 (캐릭터 그리드 `w-[228px]`, 시너지 좌측 패널 `w-[240px]`). 모바일 반응형 없음.

**영향**: 이터널 리턴 유저는 게임 중 스마트폰으로 메타를 확인하는 경우가 많음. 모바일 접근 불가 = 핵심 사용 시나리오 차단.

**해결 방향**:
- 캐릭터 그리드: 모바일에서 상단 스크롤 영역으로 전환
- 시너지: 모바일에서 스텝 형태 UX (1. 아군 선택 → 2. 결과 보기)

---

### [P1] 검색 엔진 비노출 (SEO)

**현상**: 주요 페이지가 전부 Client Component (`"use client"`). 캐릭터 분석, 시너지 페이지는 SSR 없음.

**영향**: "이터널리턴 아델 승률", "이터널리턴 파티 추천" 등 검색 유입 0. 유기적 성장 불가.

**해결 방향**:
- 캐릭터 분석 기본 통계 (현재 패치 최고 티어 기준)를 Server Component로 초기 렌더링
- `generateMetadata`로 캐릭터별 OG 태그 생성
- `sitemap.xml` 자동 생성

---

### [P1] 데스크탑 앱 Windows 전용

**현상**: Electron 빌드 타겟이 Windows EXE만 구성됨.

**영향**: Mac/Linux 사용자 배제. Steam 게임이라 윈도우 비율이 높지만, 한국 이터널 리턴 유저 중 맥북 사용자도 무시할 수 없음.

**해결 방향**: `electron-builder` 설정에 `mac`, `linux` 타겟 추가는 공수 적음. 단, 코드 서명 인증서가 추가로 필요.

---

### [P2] 듀오 기능 미완성

**현상**: ERmangho에 `/api/duo/requests` 엔드포인트 존재하나, ERMeta 프론트엔드에 해당 UI 없음. 설계만 되고 구현이 중단된 상태로 보임.

**영향**: 파티 모집/매칭 기능이 있으면 서비스 체류 시간 및 커뮤니티 형성에 큰 기여. 중단 상태가 계속되면 코드 부채화.

**해결 방향**: 기능 구현 재개 또는 완전히 제거하여 코드 정리. 반쪽짜리 기능은 유지 비용만 발생.

---

### [P2] 통계 신뢰도 표시 없음

**현상**: totalGames가 5인 조합과 5000인 조합이 동일한 방식으로 표시됨.

**영향**: 소수 데이터로 승률 100% 조합이 상위 노출될 수 있음. 사용자 신뢰도 하락.

**해결 방향**:
- totalGames < 임계값(예: 50) 조합에 "데이터 부족" 배지 표시
- 베이즈 추정을 적용한 조정 승률로 정렬 (Wilson score interval 등)

---

### [P2] 관리자 도구 부재

**현상**: `/api/admin/*` 엔드포인트는 있으나 ERMeta에 관리자 UI 없음. 패치 활성화/비활성화, 통계 재빌드를 API 직접 호출로 처리해야 함.

**영향**: 운영 효율 저하. 패치 전환 시마다 개발자 개입 필요.

**해결 방향**: 간단한 관리자 페이지 (`/admin`, 인증 미들웨어 적용). 패치 관리, 수집 상태 모니터링, 통계 재빌드 트리거.

---

### [P3] 개인화 기능 없음

**현상**: 사용자 계정 없음. 즐겨찾기 캐릭터, 최근 본 조합 등이 세션 간 유지되지 않음.

**영향**: 재방문 유인 부족. 단발성 정보 조회 후 이탈.

**해결 방향**:
- 단기: localStorage 기반 즐겨찾기 (서버리스, 구현 빠름)
- 중기: Steam 로그인 연동 (데스크탑 앱의 Steam 인증 재활용 가능)

## 7. 경쟁 포지셔닝

국내 이터널 리턴 메타 분석 서비스는 공식 통계 페이지 외에 사실상 부재. 이 서비스의 포지션:

- **단기**: "패치별 캐릭터 통계 + 파티 조합 추천"으로 고정 유저 확보
- **중기**: 데스크탑 앱의 실시간 추천 기능으로 경쟁 불가능한 진입장벽 형성
- **장기**: 커뮤니티 + 개인화로 플랫폼화

데스크탑 앱의 Player.log 기반 실시간 인식이 핵심 해자(moat). 이 기능의 완성도와 안정성을 최우선으로 높여야 한다.

---

## 8. 요약

서비스의 핵심 가치는 검증되었다: 데이터 수집 파이프라인이 동작하고, UI에 의미 있는 분석이 담겨 있으며, 데스크탑 앱이라는 차별점이 있다.

지금 단계에서 가장 급한 것은 **데이터 품질** (pickRate 0, Trio 오염)과 **접근성** (모바일, SEO)이다. 이 두 가지를 해결하면 서비스가 "볼 수 있는 것"에서 "믿을 수 있는 것"으로 도약한다.
