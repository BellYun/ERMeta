## 4. Vercel 비용 분석

### Vercel Hobby vs Pro

| 항목 | Hobby (무료) | Pro ($20 = 29,000원) |
|------|-------------|---------------------|
| Serverless 함수 타임아웃 | 10초 | 300초 |
| Cron Jobs | 2개, 1회/일 최소 간격 | 무제한, 1분 간격 가능 |
| 팀 협업 | 없음 | O |
| 대역폭 | 100 GB | 1 TB |

### 중요: Cron 실행 주기 관련

현재 ERmangho는 **5분 주기**로 Cron이 실행됨. 이를 위해서:

- **Vercel Pro 필요** → 월 29,000원 추가 비용
- 또는 **외부 Cron 서비스** 활용으로 Hobby 유지 가능

**외부 Cron 무료 대안**:

| 서비스 | 무료 한도 | 5분 주기 가능? |
|--------|---------|--------------|
| cron-job.org | 무제한 (커뮤니티 서비스) | O |
| EasyCron | 1개 무료 | O |
| Pipedream | 100 invocations/일 | X (부족) |
| GitHub Actions | private repo 500분/월 | X (부족) |

> **cron-job.org** 무료 사용으로 Vercel Hobby 유지 가능.
> 단, 외부 HTTP Cron이라 인증 처리(`CRON_SECRET`) 필수 (이미 구현됨).

## 5. 결론 및 추천

### 현재 상황 평가

**Supabase Pro 35,000원은 적절한 비용이다.**

- 데이터 볼륨 기준 (2~5 GB): 한도(8 GB)에 여유 있음
- DevOps 부담 없음: 백업, 모니터링, 튜닝 모두 Supabase 담당
- 대안 비교 시 12개월 TCO가 모든 자체 구축 옵션보다 낮음

---

### 추천 A: 현상 유지 (권장)

**월 비용: 35,000원**

Supabase Pro를 유지한다. 현재 데이터 규모에서는 가장 합리적.

```
Vercel Hobby (무료)
  + cron-job.org 무료 외부 Cron (5분 주기 HTTP 호출)
  + Supabase Pro (35,000원)
= 월 35,000원
```

만약 Vercel Pro를 사용 중이라면, cron-job.org로 전환하여 Hobby로 다운그레이드하면 월 29,000원 절약 가능.

---

### 추천 B: 최저 비용 구성 (운영 부담 감수 시)

**월 비용: 약 5,700~7,000원**

```
Hetzner CX22 VPS (€3.79/월 ≈ 5,700원)
  - PostgreSQL (self-hosted)
  - ERmangho Next.js (Docker 컨테이너)
  - systemd timer 또는 cron (5분 주기)
= 월 5,700원
```

**조건**: 백업, 모니터링, 장애 대응을 직접 처리할 의향이 있을 때.
**예상 절약**: 월 28,000원 (연 336,000원)
**초기 투자 공수**: 40~60시간 (전환 회수에 약 17~24개월)

### 단계별 마이그레이션 경로 (만약 전환 결정 시)

```
Step 1 (즉시, 무료):
  cron-job.org → Vercel Hobby 전환으로 월 29,000원 절약

Step 2 (1개월, DB만 이전):
  Supabase → Neon Pro ($19/월 ≈ 27,700원)
  - Prisma schema 그대로 사용 가능
  - pg_dump/pg_restore로 마이그레이션
  - 추가 절약: 월 7,300원

Step 3 (3개월 이상, 완전 자립):
  VPS (Hetzner) + 자체 PostgreSQL
  - Docker Compose: Next.js + PostgreSQL
  - 백업: pg_dump → S3 (Backblaze B2 무료 10GB)
  - 모니터링: Grafana Cloud 무료 tier
  - 최종 비용: 월 5,700원
```
